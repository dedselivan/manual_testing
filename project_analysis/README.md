# Анализ для планирования регрессионного тестирования

**Задача:** На основе данных о количестве багов, приоритете функций и наличии автотестов принять решение, какие функции необходимо включать в регрессионный набор тестов после нового релиза.

**Ключевой вывод:** Решение о включении в регрессию принимается не только на основе приоритета функции, но и на основе её **стабильности** (динамики багов) и **наличия инструментов для быстрой проверки** (автоматизации).
